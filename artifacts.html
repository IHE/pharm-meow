This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Logical Models

These define data models that represent the domain covered by this implementation guide in more business-friendly terms than the underlying FHIR resources.

Dosaging

Usage instructions. Instructions about requested administration of the product. Based on FHIR Dosage complex data type.
Medicinal product

Medication information. The model is shared by statements, requests, dispensations, and treatment lines. Each of those may have different restrictions in FHIR profile.
Patient

Basic patient data relevant for this use case
Practitioner

Basic prescribing/dispensing/caretaking practitioner’s data relevant for this use case

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

Medication Overview Composition

The profile for how to organize the information in a medication overview
Medication Treatment

The profile for Medication Treatment in a Medication Record - a set of treatment lines/items
Medication Treatment Line

The profile for Medication Treatment Line - a single recorded item/line of a medication treatment
Medication overview

The profile for the full Medication overview

Structures: Extension Definitions

These define constraints on FHIR data types for systems conforming to this implementation guide.

BusinessVersion

Business Version of the resource
Classification

Medication classification/category. Allows the product to be classified by various systems, e.g ATC, narcotic class, legal status of supply, etc.
MedicinalProductName

Name of the medicinal product. Optionally the type of name (Full official name, prescribing name, invented name) and language of the name can be specified. The extension is a simplified approach of what is used in MedicinalProductDefinition resource.
Substitution

Whether and which type of substitution is allowed for this medication treatment line
TreatmentStatus

The status of the actual treatment - ongoing, temporarily stopped, etc
VerificationInformation

Verification of medication line. The overview is verified as a whole, but each line is equipped with the information. The actual functional meaning and impact of this validation depends on the implementation.
